## Architecture Highlights

### Security & Compliance

**Application Security:**
- **Role-based permission middleware** with granular access controls
- **Server-side session management** with PostgreSQL (connect-pg-simple)
- **Secure password hashing** with bcrypt (10 rounds)
- **Team data isolation** with security filters on all queries
- **Object-level ACL** for fine-grained resource access control
- **SQL injection prevention** via parameterized queries (Drizzle ORM)
- **XSS protection** via React automatic escaping
- **CSRF protection** via SameSite cookie settings

**Payment & Data Security:**
- **PCI DSS Compliant** - Payment processing via Helcim (Level 1 PCI DSS)
- **Webhook signature verification** (HMAC SHA-256 for Helcim)
- **No credit card storage** - Tokenized payments only
- **TLS/SSL required** for all connections
- **Secure file uploads** via UploadThing with virus scanning

**Legal Compliance:**
- **TCPA Compliant** - Opt-out tracking, consent management, and message logging
- **CAN-SPAM Compliant** - Email opt-outs and unsubscribe links
- **Data Privacy** - User data deletion workflows on account closure
- **Audit Logging** - Complete audit trail for SMS, payments, and critical operations

**Best Practices:**
- Regular security updates and dependency patches
- Comprehensive error logging without exposing sensitive data
- Environment variable secrets (never committed to Git)
- Database encryption at rest (Neon default)
- Regular automated backups

### Data Model
- **Multi-tenant architecture** with team and program isolation
- **Canonical contact matching** prevents duplicate players via email/phone normalization
- **Comprehensive audit logging** for SMS messages, payments, and critical operations

- **Automatic database migrations** on server startup for seamless deployments

- **Query result caching** with TanStack Query for optimized data fetching
- **Optimized database queries** with Drizzle ORM and prepared statements
- **Serverless-ready** architecture with Neon PostgreSQL
- **Connection pooling** for database efficiency

### Developer Experience
- **End-to-end TypeScript** for type safety from database to UI
- **Shared schema validation** with Zod and Drizzle
- **Hot module replacement** with Vite for fast development

## Getting Started



### Prerequisites

**Required:**
- Node.js 20+ 
- PostgreSQL database (Neon recommended for serverless)

**For Full Feature Set:**
- Twilio account (SMS messaging)
- Google Cloud API key (Maps + Gemini AI)
- Firebase project (push notifications)
- SendGrid account (email)
- Helcim account (payment processing)
- UploadThing account (file uploads)
- DocuSeal account (digital signatures)



**Important Production Notes:**
- ‚úÖ Database migrations run automatically on server startup
- ‚úÖ Ensure all environment variables are set (see `.env.example`)
- ‚úÖ Configure CORS for your domain
- ‚úÖ Set `NODE_ENV=production`
- ‚úÖ Use a process manager (PM2, systemd, Docker) for reliability
- ‚úÖ Enable SSL/TLS for all endpoints (required for push notifications)
- ‚úÖ Configure SendGrid domain authentication for email deliverability
- ‚úÖ Set up Firebase project for push notifications with VAPID keys
- ‚úÖ Configure automatic backups for PostgreSQL database
- ‚úÖ Set up monitoring (error tracking, performance monitoring)

**Typical Response Times:**
- API Endpoints: 50-200ms
- Page Loads: 1-2s (first load), <500ms (cached)
- AI Box Score Processing: 10-30s
- SMS Delivery: 1-3s
- Push Notifications: <1s
